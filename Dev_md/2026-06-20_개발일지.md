# 조선대학교 AI특강 사이트 개발일지

- **기간**: 2026-06-19 ~ 2026-06-20
- **사이트**: https://chosun.dreamitbiz.com
- **저장소**: aebonlee/chosun
- **스택**: React 18 · Vite 5 · Supabase Auth · GitHub Actions → Pages(커스텀 도메인)
- **디자인 출처**: Claude Design 핸드오프 `조선대 교원 교육.dc.html`(dc-runtime) 픽셀-퍼펙트 포팅

---

## 1. 초기 구축

- `조선대 교원 교육.dc.html` 디자인을 React + Vite로 픽셀-퍼펙트 이식
  - 섹션: NAV · HERO · OVERVIEW(네이비) · CURRICULUM · LABS · PREP · INFO · FOOTER
  - 팔레트: bg `#F6F2EA`, ink `#1B1916`, navy `#1E3A5F`, terracotta `#C2603D`, border `#E2D9C9`
  - 폰트: Noto Serif KR · IBM Plex Sans KR · Newsreader
- OG 메타태그 + OG 이미지(1200×630, sharp로 SVG 렌더, 상시 의존성 아님)
- GitHub Actions → Pages 자동배포, `build_type=workflow` 전환, 커스텀 도메인 `base: '/'`

## 2. 인증

- 초기 "교육 신청 폼"(Supabase insert) → **구글/카카오 OAuth 로그인**으로 교체
  - `LoginModal`(구글·카카오) + `useAuth` 훅(`getSession` + `onAuthStateChange`)
  - 리다이렉트는 공유 Supabase 프로젝트 allow-list의 `*.dreamitbiz.com/**` 와일드카드로 커버
  - 사용하지 않게 된 `chosun_applications` 테이블 정리

## 3. 학습강의안

- 일자별·교시별 학습 콘텐츠(`lectureNotes.js`) + 좌측 사이드바 뷰(`LectureNotes.jsx`)
- 로그인 게이트(참여자 전용), 해시 라우팅(`#lecture/<id>`)
- **시간표 정정**: 50분 수업 + 10분 휴식 × **7교시**(점심 12:00–13:00), 1일 7시간/2일 14시간
  - 일자별 주제 7개로 확장(공식 핵심 주제 전부 포함 + 보충 주제)
- 본문 마침표 기준 문장별 줄바꿈(숫자 뒤 마침표는 보호)
- **문체를 학술적·기술적으로 재작성**: LLM·토큰·환각·few-shot·grounding·구성적 정렬·백워드 디자인·루브릭·재현가능성·연구윤리 등 용어 도입 및 정의 병기, 학습목표를 학술 행동동사로 통일

## 4. 프롬프트 메뉴 4종 이식 (aebonlee/claude → 조선대 디자인 재구성)

- "프롬프트" 드롭다운(가이드·연습장·갤러리) + "프롬프트 평가"를 준비사항 앞에 배치
- 해시 라우팅(`#prompt-guide/practice/gallery/eval`)
- **프롬프트 가이드**: 7개 섹션 데이터 + react-markdown/remark-gfm + 코드 하이라이터(`CodeBlock`)
- **프롬프트 평가**: 6대 기준 자동 채점(0~100점·A~D) + Before/After + 실습 과제
- 4개 모두 백엔드/API 없이 동작하는 정적 페이지

## 5. 버그 수정 및 UX 개선

- **프롬프트 평가 빈 페이지 수정**: `PromptEval.jsx`에서 `NEWS` 폰트 상수 누락 → 렌더 시 `ReferenceError`로 전체 화면이 비던 문제. SSR 스모크 렌더 테스트(`react-dom/server`)로 진단·검증
- **프롬프트 드롭다운 사용성 개선**: hover 데드존으로 하위 항목 클릭이 어렵던 문제 → `onMouseLeave` 제거 + 투명 paddingTop 브리지, 클릭 토글 + 바깥 클릭/ESC 닫기
- 로고 텍스트: "조선대학교 교원 교육" → **"조선대학교 AI특강"**
- 히어로: 문구 줄바꿈, 우측에 둥둥 뜨는 **Claude 스파크 로고(SVG, floatBob)**, 스탯 4칸 중앙정렬
- 프롬프트 평가 목표 점수 67점 → 80점
- 프롬프트 갤러리 설명 문구 수정

## 6. 학습강의안 강의·PPT 기초자료화

- 학습강의안 문체를 **학술적·기술적**으로 전면 재작성(LLM·토큰·환각·few-shot·grounding·구성적 정렬·백워드 디자인·루브릭·재현가능성·연구윤리 등 용어 도입·정의 병기, 학습목표를 학술 행동동사로 통일)
- 본문 **마침표 기준 문장별 줄바꿈**(숫자 뒤 마침표 보호)
- **14개 교시 콘텐츠 대폭 보강** — 교시마다 4요소 추가/심화:
  - 개념 심화 설명(섹션 확장)
  - **사례·예시**(`example`: 시나리오 + 입력 프롬프트 + 예상 출력)
  - **슬라이드 구성 제안**(`slides`: PPT 장표 흐름 5~8장, 제목+요점) → PPT 골격 즉시 활용
  - **강의 진행 노트**(`teachingNotes`: 도입·강조·시연·전환 등 강사 스크립트)
- `LectureNotes.jsx` 렌더러에 사례/슬라이드/진행노트 블록 추가 + `useState` window 가드(SSR 안전)
- **상단 nav 날짜별 분리**: 단일 학습강의안 링크 → `Day 1 강의안`·`Day 2 강의안` 드롭다운 2개(각 7교시 + Day1 과정개요). 프롬프트 포함 3개 드롭다운을 공용 `NavMenu` 컴포넌트로 일반화(`openMenu` 단일 상태)
- 로고 "조선대학교 교원 교육" → "조선대학교 AI특강", 히어로 문구 줄바꿈, 로그인 게이트 안내 줄바꿈

## 7. 강의안 콘텐츠 상세화 & 메뉴 단순화 (2차)

- **슬라이드 구성 상세화**: 14개 교시 `slides`를 교시당 6~8장 / 슬라이드당 3~4개 불릿으로 확장(총 87장, 표지·학습목표·도입·정리 포함)
- **상단 메뉴 Day 드롭다운 제거**: `Day 1 강의안`·`Day 2 강의안`을 드롭다운 → 단순 링크(각 일자 첫 교시로 이동), 세부 교시는 좌측 사이드바. 미사용 `dayItems`·`goLecture` 정리(프롬프트 메뉴는 `NavMenu` 드롭다운 유지)
- **강의 진행 노트 상세화**: 14개 교시 `teachingNotes`를 50분 운영 흐름(도입·개념·시연·실습·예상질문·정리)에 맞춰 시간 배분 포함 6~7항목으로 확장(총 97항목, 강사 스크립트)
- **슬라이드 섹션 제목 변경**: "슬라이드 구성 제안"(+ PPT 골격 안내문) → **"강의안 · 핵심정리"**(학습자 눈높이 안내문), 불릿 내용은 유지

## 8. 실습 모듈 상세화

- `data.js` labs 8개에 **summary·objectives·steps·promptExample·tips** 추가 — 강의안 수준의 깊이
- LABS 카드를 **클릭 시 펼침**(자세히 ▾ / 접기 ▴): 펼치면 요약 + 학습목표 + 진행 절차(번호 스텝) + 예시 프롬프트(다크 박스) + TIP 표시, 카드 테두리가 모듈 색으로 강조
- 펼친 영역은 `stopPropagation`으로 카드 닫힘과 분리(프롬프트 복사 시 닫히지 않음)

## 9. 준비사항 상세화

- `data.js` prep 4개에 **detail(상세 설명)·checklist(체크 항목)** 추가
- PREP 렌더에 상세 설명 문단 + 체크(✓) 리스트 표시(항상 노출)
- 각 항목을 일자 실습(Day 1/Day 2)과 연결하고 연구윤리·민감자료 유의사항 명시

## 10. 날짜별 PPT 생성 & 콘텐츠 학습자 관점 전환

- **날짜별 PPT(.pptx) 생성**: `scripts/generate-ppt.mjs`(pptxgenjs, 임시 설치)로 강의안 데이터 → Day1(54장)·Day2(49장) 덱 생성. 표지·교시 구분·핵심정리 슬라이드, 사이트 팔레트+맑은 고딕. `public/ppt/`에 배포, 강의안 사이드바에 "Day N 강의안 PPT 다운로드" 링크 추가. 재생성: `npm run ppt`
- **"강의 진행 노트" → "이 교시 요약"**: PPT가 강의 진행을 담당하므로, 웹 콘텐츠의 강사용 진행 스크립트(도입·강조·시연 등)를 학습자용 핵심 요점 4개로 교체. 데이터 키 `teachingNotes` → `recap`(14교시), 인트로 안내문의 옛 섹션명도 정정
- 슬라이드 섹션은 "강의안 · 핵심정리"(학습자 눈높이) 유지

## 11. 개념 도식(SVG) 전면 추가

- `Diagram.jsx`: flow·converge·grid·container·quadrant·triangle·timeline 7종 SVG 도식(사이트 팔레트, 반응형 viewBox)
- **학습강의안** intro+14교시 "한눈에 보기" 도식, **실습 모듈** 8개 펼침 카드 도식, **프롬프트 4페이지**(가이드 5요소, 연습장·갤러리 흐름, 평가 6기준→등급) 도식
- SSR 스모크로 전 도식 `<svg>` 렌더 검증

## 12. 날짜별 콘텐츠 분리 + 프롬프트 연습장 정확도 강화

- **강의안 날짜 분리**: LectureNotes 사이드바를 현재 날짜 교시만 표시(intro는 Day1), 콘텐츠 하단에 이전/다음 날짜 이동 버튼. 상단 Day1/Day2 메뉴와 동기화
- **프롬프트 연습장 자가 점검**: 가짜 미리보기 제거 → 실시간 5요소(역할·맥락·지시·형식·제약+예시) 점검 패널(✓·보강 팁·n/5·준비 완료), 프롬프트 복사 버튼, claude.ai 실행 안내. "작성→점검→복사→실행" 실습 루프 구성

## 검증 메모

- 빌드만으로 안 잡히는 런타임 오류(미정의 변수 등)는 배포 전 **SSR 스모크 렌더 테스트**로 점검하는 절차 도입
- 각 변경은 빌드 → 커밋 → 푸시 → Actions 배포 → 라이브 문자열 확인 순으로 검증
